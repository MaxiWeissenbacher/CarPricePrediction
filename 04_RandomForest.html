Now let’s create the Random Forest Model. As first approach I want to predict the price of the car with the variable “ps” and “km”. Later on there will be a model with every variable in the model.

To solve this problem, I just use the car makes which are most present in the data set (can be seen in the Data Cleaning Sheet).

Next model: Find optimal mtry value

The number of variables selected at each split is denoted by mtry in randomforest function. In this step I want to find out the optimal mtry value and the build a model again based on the results to see if I can boost the model performance.I want to get the mtry value where out of bag error is at its minimum.The model before, the number of variables selected at each split was 5.

#saved "mtry" because it takes very long to compute
#mtry <- tuneRF(train[-1],train$preis, ntreeTry=500,stepFactor=1.5,improve=0.01, trace=TRUE, plot=TRUE)
load("mtry.rda")
best.m <- mtry[mtry[, 2] == min(mtry[, 2]), 1]

Conclusion

This project showed some ways for predicting car prices or car makes. At first, the data was scraped from “autoscout24.de”, one of the biggest car selling websites in Germany. During scraping the data, there were some issues. So sometimes the computer loss its internet connection and the scraper got a connection timeout. Sometimes the computer crashed and all of the scraped data in that round got deleted. So overall, the scraping process took a lot of time and one disadvantage of the connection losses was, that there isn’t an equally number of cars for ever make and for every year. But because enough data was scraped anyways, the data was used for further investigation.

As second step, the data was cleaned and visualized, in order to get an understanding of the data. Different plots showed the price distribution, the correlation between variables and other interesting things, like how many kilometers a specific fuel type goes.

The third step was to build models. Initially it was planned to use linear regression to predict the price of a car. Unfortunately it came out, that the requirements for building a linear regression weren’t satisfied. As a reason of this, I looked after machine learning algorithms, that could do this task. As random forests can both, do regression and classification I choose to use this machine learning algorithm for prediction and classification tasks.

The results in general were very good. The best model (rf_model3) achieved an R2 value of over 0.96. So price predictions can be done very well. The classification task (predict car make) also achieved good results (R2 around 0.9). This shows that random forests are a good way to predict or classify cars.
